*The shelf at a glance, and the which-skill-for-which-type map. The agent self-activates each skill by its own description; this index is for **curation** — seeing the library and deciding what to pin. The lifecycle and the verify bar live in the Skills SOP. A skill is only `verified` after passing **lift + trigger** on a real build.*

## Candidate notes
- **taste-skill** — **overlaps `frontend-design`** (both target distinctive, non-AI-slop frontends), so it is *not* promoted on arrival. The Bugadi build is the verification: build one key page with `frontend-design`, then the same page with `taste-skill`, compare output (**lift**), and check the two don't misfire against each other when both are loaded (**trigger**). Outcome decides: replace `frontend-design`, keep both with narrowed triggers, or drop taste-skill. Note: the `SKILL.md` is **87 KB** — heavy context cost, which factors into the call. Source: third-party `taste-skill` repo (download); the other ~12 skills in that repo were **not** imported (no demand yet — pull one only if a future build needs it, e.g. `image-to-code`, `imagegen`, `brandkit`).
